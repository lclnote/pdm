# PDM: Project Development Manager

プロジェクト開発管理システム

> JP1のJob Managementに着想を得た、プロジェクト開発の工程・タスク・スケジュール統合管理システム

---

## 名称について

**PDM** = **P**roject **D**evelopment **M**anager

- プロジェクトの開発工程全体をフェーズ単位で管理し、タスクの依存関係・スケジュールを統制する
- JP1のJobnet思想を継承しつつ、ソフトウェア開発プロジェクトに特化した管理へ昇華
- 日本語呼称：**プロジェクト開発管理システム**

---

## 要件定義

### 1. プロジェクト管理
- プロジェクトの作成・編集・削除
- プロジェクト単位の進捗状況ダッシュボード（全体進捗率、遅延警告）
- プロジェクト期間（開始日～終了日）の設定
- **進捗率計算ルール**（プロジェクト・フェーズ・タスク全レベルで統一）：
  - タスク数ベース or 工数ベースをプロジェクト単位で選択可能
  - タスク数ベース: `完了タスク数 / 全タスク数 × 100`
  - 工数ベース: `完了タスクの実績工数合計 / 全タスクの見積工数合計 × 100`
  - 親タスクの進捗率は配下の末端タスク（リーフ）の進捗のみを集計（中間タスクは集計対象外）
  - 重み付け設定: タスクごとに進捗率への寄与率（重み）を個別設定可能（デフォルトは均等）

### 2. フェーズ（Phase）管理
- プロジェクトを複数のフェーズに分割（例：要件定義→設計→開発→テスト→リリース）
- フェーズの順序制御（前フェーズ完了後に次フェーズ開始 等）
- フェーズごとの進捗率表示
- **ゲート管理**: 前フェーズの全タスク完了→次フェーズ開始を申請→PM承認でゲート通過（Phase進捗会議ゲートを表現）
- **並列フェーズ対応**: フェーズごとに並行/直列を設定可能。依存関係のないフェーズは並行実行可能

### 3. タスク管理（3階層）
- タスクの階層に関わらず、**子を持つタスクの担当者はサブリーダー**、**末端タスク（リーフ）の担当者は作業者**とする
- **ルートタスク**（Phase直下）→ 子タスク有なら**サブリーダー**、無なら**作業者**
- **子タスク**（ルートタスク配下）→ 孫タスク有なら**サブリーダー**、無なら**作業者**
- **孫タスク**（末端・リーフ）→ 担当者は**作業者**
- 各階層で以下の属性を保持：
  - タスク名、説明
  - 担当者（1名）、**協力者**（複数名指定可能）
  - 見積工数 / 実績工数
  - 状態と遷移ルール：
    - **未着手**: 初期状態。全依存タスク未完了
    - **着手可能**: 全依存タスクが完了し、着手可能になった状態（自動遷移）
    - **対応中**: 担当者が作業開始（担当者のみ遷移可能）
    - **承認待ち**: 成果物提出完了、承認を待つ状態（担当者が遷移）
    - **完了**: サブリーダーまたはPMが承認（承認者のみ遷移可能）
    - **保留**: 何らかの理由で作業中断。理由と再開予定日を記録（PMのみ設定可能）
    - **中断**: タスク中止。理由と判断者を記録（PMのみ設定可能）
  - 遷移パス（矢印は遷移可能方向）：
    - **未着手 → 着手可能**: 自タスクの全依存タスク完了（システム自動）
    - **未着手 → 中断申請**: 申請者が申請→PM承認→中断
    - **着手可能 → 対応中**: 担当者が開始
    - **着手可能 → 中断申請**: 申請者が申請→PM承認→中断
    - **対応中 → 承認待ち**: 全成果物を提出（担当者が遷移）
    - **対応中 → 保留申請**: 申請者が申請→PM承認→保留
    - **対応中 → 中断申請**: 申請者が申請→PM承認→中断
    - **承認待ち → 対応中**: 承認者差戻し（サブリーダー/PM）
    - **承認待ち → 完了**: サブリーダーまたはPMが承認
    - **保留申請 → 対応中**: PM却下（申請却下、元の状態に戻る）
    - **保留申請 → 保留**: PM承認
    - **中断申請 → 対応中**: PM却下（申請却下、元の状態に戻る）
    - **中断申請 → 中断**: PM承認
    - **保留 → 着手可能**: 保留解除申請→PM承認→着手可能
    - **保留 → 中断申請**: 申請者が申請→PM承認→中断
    - **中断**: 終端状態。再開不可。理由と判断者を記録
  - 申請機能：
    - **申請者**: タスク担当者またはサブリーダーが申請可能
    - **承認者**: PMのみ承認/却下可能
    - 申請時には理由・エビデンスを添付必須
    - 申請一覧ダッシュボードで承認待ち申請をPMが確認
    - 通知機能でPMに申請発生を通知
  - 状態変更権限：担当者（自身のタスクのみ）＜サブリーダー（配下）＜PM（全タスク）

### 4. スケジュール管理
- **プロジェクト全体スケジュール**: ガントチャート表示（全フェーズ×全タスクの俯瞰）
- **フェーズスケジュール**: 選択したフェーズ内のタスクのみ表示
- **タスク単位スケジュール**: 開始日・期限日の設定
- **子タスクスケジュール**: 親タスクの期間内での日程調整
- カレンダー連携（営業日カレンダー、休日設定）

### 5. 依存関係管理（JP1のJobnet依存に相当）
- **終了-開始 (FS)**: タスクA完了後にタスクB開始（最も標準）
- **開始-開始 (SS)**: タスクA開始と同時にタスクBも開始可能
- **終了-終了 (FF)**: タスクA完了と同時にタスクBも完了
- **開始-終了 (SF)**: タスクA開始後にタスクBを完了
- 依存関係の視覚的表示（ガントチャート上の矢線）
- クリティカルパス自動算出
- 依存関係のループ検出

### 6. 可視化・UI要件
- ガントチャートビュー（メイン画面）
- フェーズ別ガントチャート
- タスク依存関係ネットワーク図
- 進捗ダッシュボード（進捗率、残日数、遅延タスク一覧）
- **検索・フィルタ機能**:
  - プロジェクト内横断検索（タスク名、担当者、状態、キーワード）
  - フェーズ・タスク階層・状態・担当者・期限範囲での複合フィルタ
  - 保存フィルタ（よく使う検索条件を保存・再利用）
- **レポート・エクスポート機能**:
  - 進捗レポート（PDF出力）
  - タスク一覧・依存関係一覧のCSVエクスポート
  - ガントチャートの画像エクスポート
  - 週次・月次レポートの自動生成・メール配信

### 7. リスク管理
- プロジェクト・フェーズ・タスクの各レベルにリスクを登録
- リスク属性：名称、説明、影響度（大/中/小）、発生確率（高/中/低）、優先度（影響度×確率で自動算出）、状態（未対応/対策中/監視中/発生/クローズ）
- リスク対応策（対策案、担当者、期限）の登録・進捗管理
- リスクが顕在化した場合の課題（Issue）への変換フロー
- プロジェクト全体のリスク一覧・ヒートマップ表示
- リスク登録に基づくスケジュールバッファの自動提案

### 8. 課題（Issue）管理
- プロジェクト・フェーズ・タスクの各レベルに課題を登録
- 課題属性：名称、説明、種別（バグ/仕様変更/要望/障害/その他）、優先度（緊急/高/中/低）、状態（未対応/対応中/解決済み/却下/クローズ）、登録者、登録日
- 課題とタスク・成果物・リスクとの関連付け（どのタスクに影響するか、どのリスクから派生したか）
- コメントスレッドによる課題の議論・進捗報告
- 担当者割当と期限設定
- 課題一覧・フィルタ・状態別カンバンボード表示
- リスクから課題への自動変換（リスク状態＝「発生」時にIssue自動作成）

### 9. 成果物管理
- 各タスク（全階層）に成果物（納品物・ドキュメント・ソースコード等）を紐付け
- 成果物の属性：名称、種別（設計書/ソース/テスト結果/報告書 等）、ファイルパス/URL、バージョン、状態（未作成/作成中/レビュー中/完了）
- 成果物のレビューステータス管理（レビューア設定、承認フロー）
- タスク完了条件として「全成果物が完了状態であること」を設定可能
- プロジェクト全体の成果物一覧・進捗ダッシュボード

### 10. ユーザー管理・認証・権限
- ユーザー登録・ログイン認証（メールアドレス＋パスワード / SSO連携）
- ロール定義と権限管理：
  - **システム管理者**: 全プロジェクト閲覧・編集、ユーザー管理、システム設定
  - **PM（プロジェクト管理者）**: 自プロジェクトの全操作、メンバー割当、フェーズ・タスク作成
  - **サブリーダー**: 配下タスクの進捗管理、成果物承認、担当者割当
  - **作業者**: 自身に割当てられたタスクの状態変更、成果物アップロード
  - **閲覧者**: プロジェクトの閲覧のみ（コメント可）
- プロジェクトごとにメンバー管理（招待・ロール割当）

### 11. 管理機能
- タスクテンプレート（よく使うフェーズ構成をテンプレート化）
- 進捗報告機能
- 通知機能（タスク開始可能通知、遅延通知、リスク警告）

## 開発計画（Phase案）

| Phase | 内容 | 成果物 |
|-------|------|--------|
| **Phase 1** | 基本モデル設計＋プロジェクト管理CRUD | ER図、プロジェクトCRUD API |
| **Phase 2** | フェーズ＋タスク（3階層）CRUD | タスクCRUD API、階層管理 |
| **Phase 3** | スケジュール管理＋ガントチャート | ガントチャート画面、日程計算ロジック |
| **Phase 4** | 依存関係管理＋クリティカルパス | 依存関係設定UI、クリティカルパス表示 |
| **Phase 5** | ダッシュボード＋レポート | 進捗ダッシュボード、レポート出力、検索・フィルタ |
| **Phase 6** | ユーザー管理＋認証＋権限 | ユーザーCRUD、ログイン、ロール権限管理 |
| **Phase 7** | リスク管理 | リスクCRUD、ヒートマップ、対応策管理 |
| **Phase 8** | 課題（Issue）管理 | Issue CRUD、カンバン、コメントスレッド、リスク連携 |
| **Phase 9** | 成果物管理＋テンプレート | 成果物CRUD、承認フロー、タスクテンプレート |

## 技術スタック案

- **Backend**: Python (FastAPI) or Go
- **Frontend**: React + TypeScript + Ganttチャートライブラリ（dhtmlx-gantt など）
- **DB**: PostgreSQL
- **API**: RESTful API
